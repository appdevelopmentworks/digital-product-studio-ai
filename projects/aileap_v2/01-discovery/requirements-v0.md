## 1. プロジェクト目的

### 1.1 ビジネスゴール(KGI)

公開後 6 ヶ月以内に、AILEAP として **月間問い合わせ件数 30 件** を達成する。
本サイトを通じて中堅企業の Web 制作案件・採用候補者からの応募を増やす。

### 1.2 主要 KPI

| 指標 | 現状 | 目標 | 計測方法 |
|---|---|---|---|
| オーガニック流入 | 約 50 UU/月 | 1,000 UU/月 | GA4 + GSC |
| GEO 経由(LLM 参照)セッション | 0 | 50 セッション/月 | UTM + llms.txt 経由トラッキング |
| 採用エントリー | 0 | 月 5 件 | フォーム CV |
| フォーム CV 率 | 未計測 | 2.5% | GA4 イベント |

### 1.3 想定読者・ターゲット

- **主読者 A**: 中堅企業(従業員 50-300 名)の経営層・経営企画
  - 課題: 既存サイト刷新、AI 活用検討、コスト最適化
  - 検索行動: 「AI Web 制作」「コーポレートサイト 刷新」「DX 中小企業」
- **主読者 B**: エンジニア中途採用候補者(2-10 年経験)
  - 課題: 次の挑戦先、技術選定、AI 活用環境
  - 検索行動: 「AILEAP 採用」「Next.js エンジニア 中途」「AI 活用 開発会社」
- **副読者**: 既存顧客(継続案件相談)

## 2. スコープ

### 2.1 対象範囲

- 案件タイプ: A1 コーポレートサイト
- 想定ページ数: 11 ページ
- 主要機能:
  - 会社概要(3 組織アーキテクチャ説明込み)

### 3.2 主要 UI 要素

  - サービス(services)

  - 採用情報(careers)
- ユーザー権限: 管理者(AILEAP) + 編集者(WMAO 引継ぎ後付与)

## 4. 非機能要件

### 4.1 パフォーマンス

- Lighthouse Performance: 90+(モバイル)
- LCP: 2.5 秒以内
- CLS: 0.1 以下
- 画像: WebP / AVIF 標準

### 4.2 アクセシビリティ

- WCAG 2.2 Level AA 準拠
- キーボードのみで全機能操作可能
- スクリーンリーダー(VoiceOver / NVDA)動作確認

### 4.3 SEO / GEO

- Lighthouse SEO: 100
- 構造化データ: Organization / WebSite / WebPage / BreadcrumbList / FAQPage / Article
- llms.txt 配置(GEO)
- メタディスクリプション 100-120 字(結論先出し)

### 4.4 セキュリティ

- HTTPS 必須(HSTS 有効化)
- フォームに reCAPTCHA / hCaptcha
- microCMS API キーの環境変数管理

### 4.5 ブラウザ・デバイス対応

- Chrome / Edge / Safari / Firefox 最新 2 バージョン
- iOS Safari / Android Chrome 最新 2 バージョン
- IE11: 対応外

## 5. デザイン要件

### 5.1 トーン・キーワード

- ブランドキーワード:
  - **AI Native**(AI 活用が前提)
  - **同等品質を半額**(差別化)
  - **WCAG 2.2 AA + GEO 標準装備**(品質シグナル)
- トーン:
  - 信頼感(根拠ある主張)
  - 先進性(モダン・洗練)
  - 親しみやすさ(専門用語の咀嚼)
  - 誠実さ(過剰約束を避ける)

### 5.2 デザインシステム

- WCAG 2.2 AA をデザイントークン段階で担保(H-4 fix の実証)
- カラーコントラスト 4.5:1 以上(通常テキスト)
- フォーカスインジケーター必須
- タッチターゲット 44×44px 以上

### 5.3 提供素材状況

詳細は `00-engagement/assets-required.yaml` 参照。

| 素材 | 状態 | 提供元 |
|---|---|---|
| ロゴ | 未確定 | 本案件で art-direction-lead 提案 |
| ブランドカラー | 暫定(#2563EB 系) | 本案件で確定 |
| 写真素材 | 不要 | AI 画像生成で代替 |
| 文章素材 | 部分提供(コンセプト) | copywriter 起草 |
