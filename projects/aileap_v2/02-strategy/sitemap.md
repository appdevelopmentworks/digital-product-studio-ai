## 1. サイト全体方針

### 1.1 サイトの目的

AILEAP の3組織アーキテクチャを言語化し、A1/A2/A3 のサービス紹介を通じて
中堅企業の Web 制作・採用候補者からのリード獲得を行う。

### 1.2 主要 KGI / KPI

| 指標 | 目標 |
|---|---|
| 月間問い合わせ件数(KGI) | 30 件 |
| オーガニック流入 | 1,000 UU/月 |
| LLM 参照セッション | 50 セッション/月 |
| 採用エントリー | 月 5 件 |

### 1.3 想定ユーザー導線

```
[検索エンジン / LLM / SNS / 直接訪問]
       ↓
   トップページ
       ↓
  ┌─────┼─────┬───────┐
  ↓     ↓     ↓       ↓
サービス  会社   実績    記事
  ↓     ↓     ↓       ↓
 詳細  3 組織  詳細  関連記事
  │     │     │      │
  └─────┴─────┴──────┘
        ↓
  [お問い合わせ / 採用エントリー]
```

## 4. URL 命名規則

- 小文字英数字・ハイフン区切り
- 例: `/services/corporate-site`, `/blog/llms-txt-guide`
- 末尾スラッシュなし
- カテゴリ階層は 2 階層まで(`/blog/category/seo-geo`)
- 動的セグメントの slug は SEO キーワードを反映

### 多言語化 URL の枠組み(将来用)

```
/ja/services    ← 日本語(現在は path prefix なしも検討)
/en/services    ← 英語(v0.3 で実装)
```

i18n の詳細は `i18n-strategy.md` 参照(本案件では枠のみ実装)。

## 6. SEO/GEO 戦略との整合

### 6.1 SEO 観点

- 3 階層構造、全ページからトップへ 3 クリック以内
- カテゴリページから個別記事への内部リンク強化
- パンくず必須(構造化データ含む)

### 6.2 GEO 観点

- `/llms.txt` ルート配置
- トップ・サービス詳細・FAQ で「100字リード結論」徹底
- FAQPage 構造化データを各サービス詳細・記事に付与
- 著者情報(Person スキーマ)を記事に付与

### 6.3 重要 SEO ページ

| ページ | SEO 優先度 | 主要 KW |
|---|---|---|
| `/services/corporate-site` | ★★★★★ | 「コーポレートサイト 制作」「AI Web 制作」 |
| `/services/mediasite` | ★★★★★ | 「メディアサイト 制作」「GEO 対応」 |
| `/blog/[slug]`(GEO 解説等) | ★★★★★ | ロングテール |
| `/about` | ★★★★ | 「AILEAP」「AI Native 制作会社」 |
