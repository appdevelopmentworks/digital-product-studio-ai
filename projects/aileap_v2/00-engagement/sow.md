## 2. 業務目的

### 2.1 サイトとしての目的

- 既存 `aileap-hazel.vercel.app` を本格的なコーポレートサイトに刷新
- AILEAP の3組織アーキテクチャ(apex / dpsai / WMAO)を言語化・訴求
- 「AI Native / 同等品質を半額 / WCAG 2.2 AA + GEO 標準装備」の差別化軸を訴求
- 採用導線の確立(エンジニア中途採用)

### 2.2 検証案件としての目的

- digital-product-studio-ai v0.2 の Phase D 検証
- 21エージェント / 28スキル / 8 hook / 21テンプレートの動作確認
- 検証で発見した課題を `validation-notes.md` → `gap-analysis-v0.2.md` に集約

---

## 3. 業務範囲

### 3.1 含まれる業務

#### Phase 1: 戦略・要件定義
- 内部キックオフ(Shin + delivery-director + ux-strategy-lead)
- 競合 Web サイト分析(5社想定: 一般的な Web 制作会社 + AI 活用を訴求する会社)
- ペルソナ設計(2-3名: 中堅企業経営層 / 採用候補者 / 既存顧客)
- サイトマップ設計(3階層・11ページ)
- コンテンツ戦略策定(4柱: 業界専門性 / ハウツー / 実績 / カルチャー)
- 要件定義書 v0 → v1

#### Phase 2: デザイン
- デザインシステム策定(WCAG 2.2 AA をトークン段階で担保)
- 主要画面デザイン(11画面)
- レスポンシブ設計
- AI 画像生成によるヒーロー素材作成

#### Phase 3: 実装
- フロントエンド(Next.js 14 App Router 想定)
- microCMS 連携(Headless CMS)
- 構造化データ(JSON-LD)
- llms.txt 配置
- メタタグ・OGP・hreflang

#### Phase 4: QA・公開
- Lighthouse 検証(Performance 90+ / Accessibility 95+ / SEO 100)
- アクセシビリティ手動検証(キーボード / スクリーンリーダー)
- 主要ブラウザ動作確認
- DNS 切替・本番公開
- GA4 / GSC 設定

#### Phase 5: 公開後 30 日 (B-C1 境界)
- SEO/GEO 初動検証レポート
- 軽微なバグ修正
- WMAO への引継ぎ準備

#### 検証作業(本案件特有)
- 各エージェントの動作ログ取得
- スキル発火の確認
- hook 動作の確認
- テンプレート利用時の不足点記録(validation-notes.md)
