## 0. 本書の目的

[gap-analysis-v0.1.md](gap-analysis-v0.1.md) B-H3 で指摘された「法務領域での弁護士介在ポイントが未明示」問題を解消する。

具体的に本書は以下を定義する:

1. **AI が法的助言を行わない原則**(責任分界)
2. **弁護士介在が必須なケース**(Decision Tree)
3. **案件タイプ別の必須法務確認項目**
4. **法務テンプレートの弁護士確認ヘッダー強制ルール**
5. **クライアントとの責任分界の伝え方**

主担当: `delivery-director`、`commercial-manager`、`client-success-lead`

---

## 1. 法務責任の基本原則

### 1.1 AI は法的助言を行わない

本組織のすべての AI エージェントは、**法的助言(リーガルアドバイス)を行わない**ことを基本原則とする。これは以下を意味する:

- 法律の解釈を断定的に提示しない
- 「これは合法/違法です」と判断しない
- 訴訟リスクの評価を AI 単独で行わない
- 契約書の最終リーガルチェックを AI 単独で行わない

**法的判断は弁護士の領域**であり、AI は弁護士確認のための材料準備までを担う。

### 1.2 AILEAP の責任範囲

| 領域 | AILEAP の責任 | 弁護士の責任 |
|---|---|---|
| 法務テンプレート提供 | ひな型作成・運用ヘッダー強制挿入 | 個別案件への適用判定・最終文言決定 |
| プライバシーポリシー設計 | 標準文言・項目網羅 | 個別事業のリスク判断・文言修正 |
| 利用規約設計 | 標準文言・項目網羅 | 同上 |
| 契約書作成 | SOW・変更注文書のひな型 | 法的拘束力のある最終文言 |
| 訴訟対応 | (対応不可) | 全責任 |
| 業界規制対応 | 一般情報の整理 | 具体的判断 |

### 1.3 クライアントへの伝え方

提案書・SOW・納品物のすべてに以下の趣旨を含める:

> 本サービスで提供する法務関連の文書(プライバシーポリシー・利用規約・特商法表記等)はひな型であり、AILEAP は法的助言を行いません。本番運用前に必ず弁護士による確認を受けてください。

## 2. 弁護士介在の Decision Tree

### 2.1 ベースライン: 弁護士確認が必須なケース

```
法務関連の文書 / 判断が発生
    ↓
[Q1] 公開・配布される文書か?
    ├─ Yes → [Q2] へ
    └─ No(社内資料のみ) → 弁護士確認は任意
    
[Q2] 法的拘束力のある文書か?
(契約書・利用規約・プライバシーポリシー等)
    ├─ Yes → 弁護士確認 必須
    └─ No → [Q3] へ
    
[Q3] 個人情報・センシティブ情報を扱うか?
    ├─ Yes → 弁護士確認 必須
    └─ No → [Q4] へ
    
[Q4] 業界規制(医療・金融・教育等)に関わるか?
    ├─ Yes → 弁護士確認 必須
    └─ No → [Q5] へ
    
[Q5] 競合他社の表現・批判を含むか?
    ├─ Yes → 弁護士確認 推奨(名誉毀損リスク)
    └─ No → 弁護士確認 任意
```

### 2.2 弁護士確認が必須な文書一覧

| 文書 | 弁護士確認 | 主担当エージェント |
|---|---|---|
| プライバシーポリシー | 必須 | copywriter(ドラフト)→ Shin → 弁護士 |
| 利用規約 | 必須 | 同上 |
| 特定商取引法表記 | 必須 | 同上 |
| Cookie 同意文言 | 必須 | 同上 |
| サブスク・課金約款 | 必須 | 同上 |
| 個人情報取扱同意書 | 必須 | 同上 |
| 運用代行契約書 | 必須(クライアント側準備か AILEAP 側準備) | commercial-manager |
| 業務委託契約書 | 必須 | 同上 |
| NDA(秘密保持契約) | 必須 | 同上 |
| SOW(Statement of Work) | 推奨(高額案件は必須) | commercial-manager |
| 変更注文書 | 推奨(累積額大時は必須) | commercial-manager |
| 著作権譲渡条項を含む文書 | 必須 | 同上 |

### 2.3 弁護士確認が必須なシチュエーション

```
[1] クライアントが個人情報を 1,000 件以上扱うサイト
[2] EC・課金機能を含むサイト
[3] 医療・金融・教育・士業・薬機法対象等の業界
[4] 行政・自治体・公的機関のサイト
[5] 子供向けコンテンツを含むサイト
[6] 国際展開(GDPR / CCPA 対応)を行うサイト
[7] 著作権・肖像権・商標が複雑な案件
[8] 競合の名指し批判を含むコンテンツ
[9] 訴訟・紛争に関する情報を含むサイト
[10] AILEAP が代理で契約締結を行うケース
```

### 2.4 弁護士確認が任意なケース

以下は弁護士確認を強制しないが、推奨はする:

- 一般的な小規模 B2B コーポレートサイト(個人情報フォームなし)
- 単純な紹介 LP(個人情報取得なし)
- メディアサイトの記事コンテンツ(著作権整理は内部で実施可能)
- AILEAP 自社プロダクトの内部仕様書

### 3.4 v0.3 / v0.4 案件タイプ(参考)

#### B 系(SaaS / 業務システム)

- 利用規約(サブスク・課金条項)
- プライバシーポリシー(データ取扱・委託)
- SLA(サービス品質保証)
- 個人情報保護法 + 業界規制(金融なら金商法等)
- 国際展開時の GDPR / CCPA

#### C 系(EC)

- 特定商取引法表記(必須)
- 返品・交換ポリシー
- 配送・支払方法表記
- 景品表示法
- 古物営業法(中古品)
- 薬機法(医薬品)

## 4. 法務テンプレートの弁護士確認ヘッダー

### 4.1 強制挿入ヘッダー

[v0.2-direction.md](v0.2-direction.md) Q4 確定事項。法務 3 テンプレート(`legal-privacy-policy.md` / `legal-terms.md` / `legal-tokushoho.md`)の冒頭に必ず以下を挿入する。

```markdown
> **⚠️ 法務確認必須**
> 本テンプレートはひな型であり、AILEAP は法的助言を行いません。
> 本番運用前に必ず弁護士による確認を受けてください。
> 本テンプレートの使用に起因する一切の損害について、
> AILEAP は責任を負いません。
```

### 4.2 ヘッダー検証の自動化

`legal-pages-check.sh` フックがデプロイ前に以下を検証する:

1. 法務 3 テンプレート(`docs/legal/privacy-policy.md` / `terms.md` / `tokushoho.md`)が存在すること
2. 各テンプレート冒頭に上記ヘッダーが含まれていること
3. ヘッダーが省略・改変されていないこと

ヘッダー欠落時はフックがエラーで停止する(`exit 1`)。

### 4.3 ヘッダー削除の禁止

エージェントは以下を禁止される:

- ❌ ヘッダーを削除する
- ❌ ヘッダーの文言を弱める(「念のため確認」等)
- ❌ ヘッダーをページ末尾に移動する
- ❌ ヘッダーをコメントアウトする(HTML コメント等で隠す)

ヘッダー削除要望がクライアントから出た場合、`delivery-director` → `studio-director` → Shin にエスカレーションし、Shin の最終承認なしに削除しない(承認することはほぼない)。

## 5. 法的判断のエスカレーション経路

### 5.1 標準フロー

```
法務関連の判断が必要(コピー作成・契約条項・利用規約等)
    ↓
copywriter / commercial-manager がドラフトを作成
    ↓
client-success-lead が「これは弁護士確認が必要か」を判定
    ↓
[必要な場合]
delivery-director → studio-director → Shin にエスカレーション
    ↓
Shin が弁護士に依頼(本組織外の作業)
    ↓
弁護士確認結果を反映 → 案件に組込
    ↓
[不要な場合]
そのまま納品
```

### 5.2 エスカレーション時の必須情報

`delivery-director` がエスカレーション時に Shin に提示する情報:

```yaml
legal_escalation:
  project_id: <id>
  artifact_path: <ファイルパス>
  artifact_type: privacy_policy | terms | sow | other
  reason_for_escalation: |
    本案件は EC を含むため、特商法 + 個人情報保護法の両方の観点で
    弁護士確認が必須。Decision Tree Q3-Q4 が Yes に該当。
  client_industry: <業界>
  data_handled:
    - 個人情報(氏名・住所・電話・メール・購入履歴)
    - クレジットカード情報(決済代行経由)
  international: false
  similar_past_cases: <参考案件 ID>
  recommended_lawyer: <顧問弁護士・指定弁護士>
  urgency: high | medium | low
  budget_for_legal_review: 50000-100000  # 弁護士費用見積
```

### 5.3 弁護士費用の取扱

| ケース | 費用負担 |
|---|---|
| 標準的なプライバシーポリシー確認(クライアント案件) | クライアント負担(見積に含む) |
| EC・複雑な業界規制対応 | クライアント負担(見積に含む) |
| 自社プロダクト案件 | AILEAP 負担 |
| AILEAP 自身の契約書(SOW・運用代行契約) | AILEAP 負担 |
| 訴訟対応 | クライアント負担 + AILEAP 関与時はその責任分担 |

弁護士費用は `commercial-manager` が見積時に「実費パススルー項目」として明示する。

## 6. クライアントへの責任分界の伝え方

### 6.1 提案書での明示

`proposal-deck.md` テンプレートには以下のセクションを含める:

```markdown
## 法務関連サービスの責任分界

AILEAP が提供する以下の文書はひな型・参考情報であり、法的助言ではありません。

- プライバシーポリシー
- 利用規約
- 特定商取引法表記
- 個人情報取扱同意

これらは弁護士による最終確認を経て公開・運用してください。
弁護士は AILEAP からご紹介可能です(別途実費)。
```

### 6.2 SOW での明示

`sow.md` テンプレートには以下の条項を含める:

```markdown
### 第○条(法務関連の責任分界)

本契約に基づき AILEAP が提供する法務関連文書(プライバシーポリシー・
利用規約・特定商取引法表記・各種同意文書等)は、ひな型および参考情報
として提供するものであり、法的助言ではない。

クライアントは本番運用前に弁護士による確認を経るものとし、AILEAP は
本テンプレートの使用に起因する法的紛争・損害について一切の責任を
負わないものとする。
```

### 6.3 納品時の説明

`handoff-package.md` および `cms-manual.md` で以下を含める:

```markdown
## 法務関連文書の取扱注意

本納品物に含まれる以下の文書は弁護士確認を経たひな型ですが、
事業運営に応じて修正が必要な場合があります。

- プライバシーポリシー
- 利用規約
- 特定商取引法表記

修正時は必ず弁護士に再確認を依頼してください。
AILEAP では法的助言は提供できません。
```

## 7. 著作権・知的財産の取扱

### 7.1 制作物の著作権

| 制作物 | 著作権の帰属 | SOW での明示 |
|---|---|---|
| ロゴ・ブランド要素(クライアント支給) | クライアント | (前提) |
| AILEAP がデザインしたビジュアル | 検収後にクライアントへ譲渡 | 必須 |
| AILEAP が書いたコピー | 検収後にクライアントへ譲渡 | 必須 |
| 撮影写真(クライアント側手配) | クライアント | (前提) |
| AILEAP が手配した撮影写真 | 撮影者 → AILEAP → クライアント譲渡 | 必須(撮影契約) |
| ソースコード | 検収後にクライアントへ譲渡 | 必須(オープンソース部分は OSS ライセンスに従う) |
| AILEAP のフレームワーク・テンプレート(再利用部分) | AILEAP に保持 | 必須(クライアントへは利用許諾のみ) |

### 7.2 著作権譲渡条項の SOW テンプレート

```markdown
### 第○条(著作権の帰属)

1. 本契約に基づき AILEAP が制作する成果物の著作権(著作権法第 27 条
   および第 28 条に定める権利を含む)は、検収完了および対価の完済を
   もってクライアントに譲渡される。

2. ただし以下は本譲渡の対象外とし、AILEAP に留保される:
   - AILEAP が独自に開発・保有するフレームワーク・ライブラリ・
     テンプレート(以下「AILEAP 既存資産」)
   - 第三者が著作権を有するオープンソースソフトウェア
   - クライアントが支給した素材
   
3. AILEAP 既存資産については、クライアントに非独占的・無償の利用
   許諾を付与する。再配布・改変は禁止する。

4. AILEAP は本成果物を AILEAP の事例として紹介する権利を留保する。
   ただし NDA 案件はこの限りでない。
```

### 7.3 第三者素材の取扱

`asset-rights-validator`(v0.4 以降に正式実装。v0.2 では copywriter / ui-designer が代行)が以下を検証:

- フォントライセンス(商用利用可・Webfont 配信可)
- 写真素材(購入元 + ライセンス範囲)
- 動画素材(同上)
- アイコン・イラスト(同上)
- 音楽素材(同上)

ライセンス不明の素材は使用禁止。クライアント支給でも、ライセンス確認できない場合は使用しない。

## 8. プライバシー・個人情報保護

### 8.1 個人情報保護法対応の標準項目

`legal-privacy-policy.md` テンプレートに含めるべき項目:

```
1. 事業者情報(会社名・住所・代表者・連絡先)
2. 取得する個人情報の項目
3. 取得の方法
4. 利用目的
5. 第三者提供の有無
6. 委託の有無
7. 共同利用の有無
8. 安全管理措置
9. 開示・訂正・削除の手続
10. お問合せ窓口
11. 改定履歴
```

### 8.2 Cookie 同意 UI の標準

GA4 / GTM / Hotjar 等の計測ツールを使用する全サイトで Cookie 同意 UI を実装する。

```
標準実装:
  - 初回訪問時にバナー表示
  - 「同意」「拒否」「設定」の 3 択
  - 「設定」では分類別(必須/分析/広告)に選択可能
  - 同意状態を Cookie / localStorage に保存
  - 同意なしでは GA4 等のスクリプトを発火させない(Consent Mode v2 対応)
```

実装は `frontend-engineer` が `cookie-consent-banner` コンポーネントとして提供。

### 8.3 個人情報取扱同意の標準フロー

お問合せフォーム・資料請求フォーム等で個人情報を取得する際:

```
[1] フォーム下部に個人情報取扱の同意チェックボックス(デフォルト未チェック)
[2] チェックボックスのラベルにプライバシーポリシーへのリンク
[3] チェックなしの場合、submit ボタンが disabled
[4] submit 時にバックエンドでも同意状態を再検証
[5] 同意ログを保存(訴訟対応のため)
```

## 9. 国際展開時の追加対応(参考・v0.4 以降)

### 9.1 GDPR(EU)

EU 居住者が対象に含まれる場合:

- 明示的な同意取得(opt-in)
- データ主体権利(アクセス・訂正・削除・ポータビリティ)対応
- DPO(Data Protection Officer)指名検討
- データ越境移転の根拠明示
- 違反時の通知体制(72 時間以内)

### 9.2 CCPA(カリフォルニア州)

カリフォルニア州居住者が対象に含まれる場合:

- 「Do Not Sell My Personal Information」リンク
- 個人情報カテゴリの開示
- データ主体権利対応

### 9.3 PIPL(中国)

中国居住者が対象に含まれる場合:

- 中国国内のデータ保管要件
- 越境移転の事前申請
- 専任窓口の設置

v0.4 以降の B 系・国際展開案件で本格対応。v0.2 段階では国内案件中心のため、本書では参考レベルで記載。

## 10. 緊急時(法的紛争・問合せ)の対応

### 10.1 クライアントから法的問題の問合せが来た場合

```
[1] client-success-lead が問合せを受領
[2] 即座に delivery-director に通知
[3] AI による回答は禁止(法的判断回避)
[4] delivery-director → studio-director → Shin にエスカレーション
[5] Shin が弁護士に確認後、クライアントに回答
[6] 対応記録を decisions.yaml に記録
```

### 10.2 第三者から著作権侵害等の警告が来た場合

```
[1] client-success-lead が受領
[2] 即座に Shin にエスカレーション(他のエージェント介在を最小化)
[3] AI による回答は禁止
[4] Shin が弁護士確認後、対応(警告内容により異なる)
[5] 必要に応じて該当コンテンツの非公開化
[6] 対応記録を decisions.yaml + Shin の個人記録に保存
```

### 10.3 訴訟・紛争発生時

すべての訴訟・紛争対応は Shin + 弁護士が一切を担当。AI エージェントは関与しない。

ただし以下は AI が支援可能:

- 関連 artifacts の収集・整理
- 議事録・decisions.yaml の検索
- 過去のクライアント対応履歴の整理
